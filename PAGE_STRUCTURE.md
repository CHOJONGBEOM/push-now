## 🧭 네비게이션 디자인

### 최종 구조

```
┌────────────────────────────────────────────────────────────┐
│ 💎 PushNow                                    [프로필 👤]   │
├────────────────────────────────────────────────────────────┤
│  추천   |   피드   |   타이밍   |   트렌드   |   검토       │
│ Generate  Feed     Timing      Trends      Review          │
└────────────────────────────────────────────────────────────┘
```

### 메뉴 설명

| 메뉴명 | 영문 | 의미 | 사용자 질문 |
|--------|------|------|------------|
| **추천** | Generate | AI 메시지 자동 생성 | "뭘 보낼지 모르겠어" |
| **피드** | Feed | 실시간 경쟁사 푸시 | "경쟁사 뭐 하지?" |
| **타이밍** | Timing | 최적 발송 시간 분석 | "언제 보내지?" |
| **트렌드** | Trends | 키워드 분석 | "뭐라고 쓰지?" |
| **검토** | Review | 내 메시지 평가 | "이거 괜찮은지?" |

### 프로필 드롭다운

```
[프로필 👤] 클릭 시:
├─ 내 정보
├─ 앱 추가 요청 (Contact)
└─ 로그아웃
```

**Contact를 프로필 내부에 둔 이유:**
- 덜 자주 사용하는 기능
- 메인 네비게이션 간결하게 유지
- 사용자 관련 기능으로 그룹화

**우선순위**: ⭐⭐⭐⭐⭐ (Phase 1)  
**핵심 가치**: 즉각적인 실용성 - "고민 없이 바로 메시지 받기"

- 4단계 스텝 인터페이스 (wizard UI)
- Gemini API 연동 메시지 생성
- 경쟁사 데이터 기반 시간 자동 추천
- 원클릭 검토 페이지 이동
- 생성 히스토리 저장 (optional)

#### 데이터 소스
- **입력**: 카테고리, 목적, 시간대, 톤
- **활용**: Supabase 경쟁사 푸시 데이터, 트렌드 키워드
- **출력**: AI 생성 제목/본문, 추천 시간

- 시간대별 히트맵 시각화 (Recharts)

- 카테고리별 필터
- 추천 시간 TOP 3 알고리즘
  - 경쟁 밀도 계산
  - 사용자 활성도 고려 (optional)
- 피해야 할 시간대 경고

#### 데이터 소스
- **집계**: `posted_at` → 요일별, 시간대별 카운트
- **쿼리 예시**:
  ```sql
  SELECT 
    EXTRACT(DOW FROM posted_at) as day_of_week,
    EXTRACT(HOUR FROM posted_at) as hour,
    COUNT(*) as count
  FROM push_messages
  WHERE posted_at > NOW() - INTERVAL '7 days'
    AND category = '패션'
  GROUP BY day_of_week, hour
  ```

## 🔄 사용자 여정 (User Journey)

### 신규 사용자 플로우

```
1. 랜딩 페이지 (design/landing_page.html 스타일)
   ↓ "무료로 시작하기" 클릭
   
2. 회원가입
   ↓
   
3. 첫 페이지: AI 메시지 추천 (/generate)
   "바로 써먹을 수 있는 메시지 받기" → 즉각적 만족
   ↓ "피드 보기" 클릭
   
4. 실시간 피드 (/feed)
   "경쟁사가 뭘 하는지 확인" → 호기심 충족
   ↓ "언제 보내지?" 고민
   
5. 발송 타이밍 (/timing)
   "최적 시간 확인" → 데이터 기반 의사결정
   ↓
   
6. 정기적 사용 (Daily Workflow)
   - 아침: 피드 체크
   - 캠페인 기획 시: 추천 + 타이밍
   - 발송 전: 검토
```
