# PushNow - 비즈니스 개요

## 한 줄 정의

> **"경쟁사 푸시 알림을 실시간 수집·분석하여, 마케터에게 '언제 보낼지'와 '뭐라고 쓸지'를 데이터로 알려주는 인텔리전스 대시보드"**

---

## 왜 이 서비스가 필요한가?

### CRM 마케터의 현실 (Daily Pain Points)

| 상황 | 현재 해결 방식 | 문제점 |
|------|---------------|--------|
| "내일 푸시 언제 보내지?" | 감으로 결정, 또는 과거 데이터 참고 | 경쟁사 발송 시간은 모름 → 같은 시간대에 몰리면 묻힘 |
| "경쟁사 뭐 하고 있지?" | 폰에 경쟁 앱 깔아두고 수동 체크 | 놓치는 게 많음, 기록이 안 남음, 시간 낭비 |
| "할인 카피 어떻게 쓰지?" | 벤치마킹 레퍼런스 수집 | 체계적 분석 불가, 트렌드 파악 어려움 |
| "우리 푸시 성과가 왜 낮지?" | CTR만 보고 원인 추측 | 시장 대비 비교 기준 없음 |

### 핵심 인사이트

```
마케터는 "데이터"를 원하는 게 아니라 "답"을 원한다.
- ❌ "경쟁사가 하루 평균 3.2건 발송합니다"
- ✅ "화요일 오전 10시는 경쟁사 발송이 40% 적습니다. 이 시간에 보내세요."
```

## MVP 핵심 기능 3가지

### 1. 최적 발송 시간 추천 (Golden Hour Finder)

| 항목 | 내용 |
|------|------|
| **User Story** | "내일 프로모션 푸시를 보내려는데, 경쟁사 푸시가 쏟아지는 시간은 피하고 싶어. 언제 보내면 우리 메시지가 묻히지 않을까?" |
| **Input Data** | `posted_at` (발송 시각), `app_name`, `category` - 최근 7~30일 |
| **Output** | 시간대별 히트맵 (0~23시 × 요일) + 추천 시간대 TOP 3 + 인사이트 문구 |
| **핵심 가치** | "언제 보내야 하나?" 질문에 데이터로 답변 |

**화면 예시**
```
┌─────────────────────────────────────────────┐
│  🎯 추천 발송 시간                           │
│                                             │
│  1위: 화요일 10:00  (경쟁 밀도 -42%)         │
│  2위: 목요일 14:00  (경쟁 밀도 -38%)         │
│  3위: 일요일 20:00  (경쟁 밀도 -35%)         │
│                                             │
│  ⚠️ 피해야 할 시간: 금요일 12:00 (최고 밀집) │
└─────────────────────────────────────────────┘
```

---

### 2. 실시간 경쟁사 푸시 피드 (Live Feed)

| 항목 | 내용 |
|------|------|
| **User Story** | "경쟁사가 갑자기 50% 할인 프로모션을 시작했는지 바로 알고 싶어. 우리도 빠르게 대응해야 하니까." |
| **Input Data** | `app_name`, `title`, `content`, `category`, `posted_at` - 실시간/24h |
| **Output** | 타임라인 피드 (앱 아이콘 + 메시지 카드) + 앱/카테고리 필터 + 오늘 발송량 |
| **핵심 가치** | "경쟁사 뭐 하는지 한눈에" - 가장 직관적인 가치 |

**화면 예시**
```
┌─────────────────────────────────────────────┐
│  📡 실시간 피드                    오늘 47건 │
├─────────────────────────────────────────────┤
│  [무신사]              방금 전               │
│  🏷️ SALE                                    │
│  "단 3일! 인기 브랜드 최대 70% OFF"          │
├─────────────────────────────────────────────┤
│  [29CM]                12분 전              │
│  🆕 NEW                                     │
│  "2월 신상 컬렉션이 도착했어요"              │
├─────────────────────────────────────────────┤
│  [지그재그]            34분 전              │
│  🎁 EVENT                                   │
│  "오늘만 무료배송 쿠폰 🎉"                   │
└─────────────────────────────────────────────┘
```

---

### 3. 트렌드 키워드 & 카피 분석 (Copy Intelligence)

| 항목 | 내용 |
|------|------|
| **User Story** | "요즘 경쟁사들이 어떤 키워드를 많이 쓰지? '무료배송'이 대세인지 '한정수량'이 대세인지 알면 우리 카피도 참고할 수 있을 텐데." |
| **Input Data** | `title`, `content` → 키워드 추출, `posted_at`, `category` |
| **Output** | 주간 인기 키워드 TOP 10 + 키워드 추이 그래프 + 실제 푸시 예시 |
| **핵심 가치** | 카피라이팅 벤치마킹 - "남들이 뭐라고 쓰는지" |

**화면 예시**
```
┌─────────────────────────────────────────────┐
│  🔥 이번 주 인기 키워드                      │
├─────────────────────────────────────────────┤
│  1. 무료배송 (142회) ████████████ +23%      │
│  2. 한정수량 (98회)  ████████░░░░ +15%      │
│  3. 단독특가 (87회)  ███████░░░░ -5%        │
│  4. 오늘만  (76회)   ██████░░░░░ +8%        │
│  5. 신상    (71회)   █████░░░░░░ +31%       │
├─────────────────────────────────────────────┤
│  💡 "무료배송" 포함 푸시 예시 보기 →         │
└─────────────────────────────────────────────┘
```

## 경쟁 환경 분석

### 직접 경쟁

| 서비스 | 특징 | PushNow 차별점 |
|--------|------|---------------|
| **SimilarWeb** | 웹 트래픽 분석 중심 | 푸시 알림 특화 X |
| **App Annie (data.ai)** | 앱스토어 순위/리뷰 | 푸시 메시지 수집 X |
| **Mobile Action** | ASO 중심 | CRM 마케터 타겟 X |

### 간접 경쟁 (마케터가 현재 하는 방식)

| 방식 | 한계 |
|------|------|
| 경쟁 앱 직접 설치 | 수동, 놓치는 게 많음, 기록 안 됨 |
| 스크린샷 수집 | 체계적 분석 불가, 검색 불가 |
| 팀 슬랙에 공유 | 산발적, 트렌드 파악 어려움 |

## 사용 시나리오

### 시나리오 1: 주간 푸시 캠페인 기획

```
월요일 오전, 김마케 씨의 업무 시작

1. PushNow 접속 → 실시간 피드 확인
   "어, 무신사가 주말에 대규모 할인했네. 우리도 대응해야겠다."

2. 시간 분석 탭 → 히트맵 확인
   "화요일 오전 10시가 경쟁 밀도 낮네. 여기로 잡자."

3. 키워드 분석 탭 → 트렌드 확인
   "'무료배송' 키워드가 급상승. 우리 카피에도 넣어볼까?"

4. 캠페인 기획 완료 → 팀 공유
```

### 시나리오 2: 긴급 경쟁사 대응

```
금요일 오후 3시, 알림 수신

"[무신사] 오늘부터 72시간 한정 최대 80% 세일"

1. PushNow 피드에서 즉시 확인
2. 상세 내용 파악 (카테고리, 할인율, 기간)
3. 우리 팀에 슬랙 공유 + 대응 논의
4. 30분 내 우리도 푸시 발송 결정
```

## 기술 스택

| 영역 | 기술 | 선택 이유 |
|------|------|----------|
| **프론트엔드** | React 19 + TypeScript | 최신 버전, 타입 안정성 |

| **배포** | Vercel | Git 연동 자동 배포, 무료 |
