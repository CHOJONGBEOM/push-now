## 1. 데이터 마이닝 및 수집 아키텍처

PushNow의 데이터 수집은 두 가지 독립된 경로(Android 실물 기기 및 로컬 에뮬레이터)를 통해 작동하며, 최종적으로 Supabase 클라우드 데이터베이스로 정규화되어 적재됩니다.

### 1.1 Android 알림 수집기 (`push_radar_app`)
* **위치**: [push_radar_app](file:///c:/workspace/push_radar_app)
* **메커니즘**:
  * React Native 모듈 `react-native-android-notification-listener` 기반으로 작동하는 백그라운드 리스너 서비스.
  * 감지된 알림은 패키지 필터링(사용자가 앱 내 `SettingsScreen`에서 직접 설정한 활성 앱 목록) 및 차단 패키지(`com.kakao.talk`, `android` 등 시스템/개인정보 앱) 필터를 거친 뒤, `posted_at` 시간대를 UTC로 변환하여 Supabase REST API로 즉시 전송됩니다.
  * 카테고리(`category`)는 `null`로 전송되며, Supabase 서버 측에서 AI 기반으로 자동 분류됩니다.

### 1.2 Python 백그라운드 수집기 (`push_msg_collector`)
* **위치**: `c:\workspace\push_msg_collector`
* **메커니즘**:
  * 24시간 가동되는 Android 에뮬레이터(AVD) 환경에서 10초마다 `adb shell dumpsys notification`을 실행하여 알림 덤프 텍스트를 수집.
  * Python 기반 파서(`parser.py`)가 정규식을 통해 앱 이름, 알림 제목, 본문을 분리하고 Supabase REST API로 전송.

## 2. Supabase 데이터베이스 스키마

### 2.1 `push_messages` 테이블 스키마 DDL
```sql
CREATE TABLE public.push_messages (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  package_name TEXT NOT NULL,                  -- 앱 패키지명 (예: com.musinsa.store)
  app_name TEXT NOT NULL,                      -- 앱 한글명 (예: 무신사)
  title TEXT,                                  -- 푸시 제목
  body TEXT NOT NULL,                          -- 푸시 본문
  posted_at TIMESTAMPTZ NOT NULL,              -- 알림 발송 시각
  collected_at TIMESTAMPTZ DEFAULT NOW(),      -- DB 수집 시각
  is_ad BOOLEAN DEFAULT FALSE,                 -- 광고성 여부
  category TEXT,                               -- 카테고리 (체크 제약 조건 참조)
  has_emoji BOOLEAN DEFAULT FALSE,             -- 이모지 포함 여부
  message_length INTEGER,                      -- 메시지 글자수
  is_hidden BOOLEAN DEFAULT FALSE,             -- 대시보드 숨김 플래그
  raw_data TEXT,                               -- Android 알림 JSON 원본
  
  -- AI 분석 확장 필드
  marketing_hook TEXT,                         -- AI가 추출한 핵심 마케팅 훅 (최대 15자)
  hook_type TEXT,                              -- 훅 종류 (체크 제약 조건 참조)
  hook_trigger TEXT,                           -- 심리 트리거 (체크 제약 조건 참조)
  hook_analyzed_at TIMESTAMPTZ,                -- AI 분석 완료 시간
  device_id TEXT,                              -- 수집한 디바이스 고유 ID

  -- 유니크 제약조건 (중복 수집 방지)
  CONSTRAINT unique_push UNIQUE (package_name, body, posted_at, device_id),
  
  -- 체크 제약조건 (올바른 데이터 카테고리 규격 제안)
  CONSTRAINT valid_hook_type CHECK (hook_type IN ('price', 'urgency', 'personal', 'curiosity', 'newness', 'social_proof', 'benefit', 'content', 'event', 'community', 'other')),
  CONSTRAINT valid_hook_trigger CHECK (hook_trigger IN ('scarcity', 'greed', 'personalization', 'curiosity', 'social_proof', 'novelty', 'none'))
);
```

### 2.2 인덱스
```sql
CREATE INDEX idx_push_app_time ON push_messages (app_name, posted_at DESC);
CREATE INDEX idx_posted_at ON push_messages (posted_at DESC);
CREATE INDEX idx_is_hidden ON push_messages (is_hidden) WHERE is_hidden = true;
```

## 3. 프론트엔드 TypeScript 타입 정의

대시보드 앱에서 사용하는 핵심 데이터 규격 정의는 [src/types/push-message.ts](file:///c:/workspace/push_now/src/types/push-message.ts)에 위치합니다.

export interface PushMessage {
  id: number;
  package_name: string;
  app_name: string;
  title: string | null;
  body: string;
  posted_at: string;
  collected_at: string;
  is_ad: boolean;
  category: 'promo' | 'product' | 'retention' | 'system' | 'transaction' | 'empty' | 'other' | null;
  has_emoji: boolean;
  message_length: number;
  is_hidden: boolean;
  
  // AI 확장 필드

  hook_analyzed_at: string | null;
  device_id: string | null;
